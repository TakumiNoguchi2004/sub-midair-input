# 日本語入力 (折り曲げ式) — 現時点の仕様

カメラの手検出 (MediaPipe) を使った、空中での日本語かな入力。**フロント完結** (backend 不要) で、
実装は `packages/web/src/midair_web/static/js/modes/japanese.js`。設定 UI も同ファイル
(`renderJapaneseSettings`)、横断的な安定化処理は `core.js`。

> ⚠️ ここで扱うのは「かなを **入力** する」まで. 候補生成・辞書・かな漢字変換・ランキング・
> CLI 統合は未実装 (親 Issue #17)。

---

## 1. 入力の流れ

「**指を折って行(子音)を選び、手を動かして母音を選ぶ**」フリック入力。基準はパー(全指を開いた状態)。

```
パー(基準)
  → 指定の指を折る(それ以外は伸ばす)         … 行(子音)を選択
  → その姿勢を HOLD_MS 保持                   … 行をロック(確定)
  → 手全体を FLICK_DIST 以上動かす            … 母音(方向)を1つロック
  → パーに戻す                               … 確定して出力
```

- **フリックせずにパーへ戻す = 「あ段」**(中央)。
- 折り曲げ・フリックは **各入力で1回だけロック**される (ロック後は指やフリックが多少ぶれても変わらない → 誤爆・ちらつき防止)。
- 1 文字ごとにパーへ戻す (パーが次の入力の開始基準)。

---

## 2. 運指マッピング (行/操作 → 折る指)

指の記号: **親**=親指, **人**=人差し指, **中**=中指, **薬**=薬指, **小**=小指。
「折る指」以外はすべて伸ばしているのが条件。

| 行 / 操作 | 折る指 (既定) | 出力 / 動作 |
|---|---|---|
| あ行 | 親 | あ い う え お |
| か行 | 人 | か き く け こ |
| さ行 | 中 | さ し す せ そ |
| た行 | 親+人 | た ち つ て と |
| な行 | 親+中 | な に ぬ ね の |
| は行 | 親+薬 | は ひ ふ へ ほ |
| ま行 | 人+中 | ま み む め も |
| や行 | 中+薬 | や （ ゆ ） よ |
| ら行 | 親+人+中 | ら り る れ ろ |
| わ行 | 人+中+薬 | わ を ん ー 〜 |
| **濁点** | 親+薬+小 | 直前文字を 濁音→半濁音→小→元 でループ (フリック無し) |
| **消す** | 親+中+薬+小 (=人差し指のみ立てる) | 直前 1 文字を削除 (フリック無し) |

運指は **設定 UI で変更可能** (下記 5.)。既定値は `japanese.js` の `DEFAULT_FOLD`。

---

## 3. 母音 / フリック方向

行を選んだあと、手を動かした方向で母音(段)が決まる。

| 方向 | 母音(段) | あ行での例 |
|---|---|---|
| 中央 (動かさない) | あ段 | あ |
| 左 | い段 | い |
| 上 | う段 | う |
| 右 | え段 | え |
| 下 | お段 | お |

行によって割り当てが特殊なもの:

- **や行**: 中央=や / 左=（ / 上=ゆ / 右=） / 下=よ
- **わ行**: 中央=わ / 左=を / 上=ん / 右=ー / 下=〜

## 4. パラメータ (しきい値)

すべて設定 UI のスライダで**その場で調整可能**。既定値と意味:

| パラメータ | 既定 | スライダ範囲 | 意味 |
|---|---|---|---|
| 行ロック `HOLD_MS` | **150** ms | 60–600 | 折りポーズをこの時間保持したら行を確定ロックする。短いと素早いが誤爆しやすい。長いと安定するが待たされる。 |
| フリック距離 `FLICK_DIST` | **0.16** | 0.02–1.00 | 手のひら基準点が画角のこの割合だけ動いたら「フリックした」とみなす (正規化 0〜1)。大きいほど大きく動かす必要 = 誤フリックが減るが疲れる。 |
| 親指折りしきい `THUMB_FOLD` | **0.6** | 0.40–1.10 | 親指を「折った」と判定する近さ (親指先と中指付け根の距離 ÷ 手の大きさ)。大きいほど折りと判定されやすい。**環境差が出やすい最重要調整項目**。 |

---

## 5. 設定 UI (日本語モードの右パネル)

日本語モードにすると、右カラムに設定パネルが出る。

- **検出しきい値スライダ** (3つ): 上記パラメータをリアルタイムに変更。
- **運指エディタ**: 行/濁点/消す ごとに、折る指を `✅` で表示。
  - 「**運指変更**」を押すとチェックボックスで編集でき、「**保存**」で確定。
  - 保存時に **他のエントリ(行/濁点/消す)と運指が重複していないか検証**し、重複していれば却下 (「他の運指と重複」)。1 本も選ばない場合も却下。
- 変更したしきい値・運指は **入力言語を切り替えて戻っても維持**される (モジュール変数。ページ再読込でリセット)。

## 6. 検出の仕組み (内部)

- **指の折り判定**
  - 人/中/薬/小: 指先が第2関節(PIP)より手首に近ければ「折り」(伸展判定 `fingerUp` の否定)。
  - 親指: 親指先が中指の付け根に近い (`dist(親先, 中付根) / 手の大きさ < THUMB_FOLD`) なら「折り」。
- **手の基準点 `palmPoint`**: 手首 + 人差し/小指の付け根の中心。**折り曲げでは動かない点**なので、指を折る動作自体が移動として誤検出されない。
- **フリック**: 折り曲げ開始時の `palmPoint` を起点に、以後の移動量が `FLICK_DIST` を超えた瞬間の方向を採用。`|dx|>|dy|` なら左右、そうでなければ上下。一度採用したら変えない。
- **濁点**: 直前文字を対応表 (`DAKUTEN_CYCLES`) で 濁音→半濁音→小→元 に巡回。対応形が無い字はスキップ。
- 座標は自撮りミラー基準 (`1 - x`) で扱い、画面の左右と手の左右が一致する。

## 7. 前提となる横断的な安定化 (`core.js`)

日本語入力を安定させるため、全モード共通で以下が効いている:

- **言語切替は「手を開いている(パー相当=4本指伸展)」ときだけ評価**。指を折って入力している間は言語切替が起きない (逆に、言語切替モーション中=手の甲は入力を止める、と対称)。
- **手の一部しか写っていないフレーム(手のひら見切れ等)は無効化**。見切れたランドマークによる誤検出・言語切替連発を防ぐ。
- 描画/入力座標は画角の内側 (`EDGE_MARGIN`) にリマップ+クランプ。端の不安定を回避。

---

## 8. 調整の指針 / 既知の制限

- まず **親指折りしきい (`THUMB_FOLD`)** を合わせる。あ/た/な/は/ら/濁点/消す が親指に依存するため、ここがずれると多くの入力が崩れる。
- 反応が早すぎて誤爆する → `HOLD_MS` を上げる。フリックが暴発/不発 → `FLICK_DIST` を調整。
- 親指の折り検出はヒューリスティックで、カメラ角度・手の個体差の影響を受けやすい (今後の改善余地)。
- くり返しになるが、本機能は「かな入力」まで。変換・候補・ランキング・辞書は未実装。
